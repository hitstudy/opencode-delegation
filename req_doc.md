## 文档 1：需求文档

```markdown
# Subagent Launcher — 需求文档

## 1. 背景

在使用 AI 编程助手执行长周期任务时，存在严重的资源浪费问题：

- 强模型（如 Claude Opus/Sonnet）持续消耗高价值 token 在低认知密度工作上
- 长输出（测试日志、构建日志、监控流）污染主模型上下文窗口
- 主模型被迫反复读取和理解大段重复性输出
- 用户需要长时间等待，期间模型仍在"思考"但没有产生高价值输出

典型场景：运行测试套件、等待构建完成、监控日志流、批量文本处理、信息筛选汇总。

这些任务的共同特征是：**执行阶段不需要高智力，只需要稳定执行和异常检测**。

## 2. 目标

### 2.1 核心目标

| 目标 | 衡量标准 |
|------|----------|
| 减少 token 浪费 | 委托阶段主模型 token 消耗降为零（前台模式） |
| 减少上下文污染 | subagent 输出经过滤后返回，不污染主模型上下文 |
| 提升长任务稳定性 | subagent 独立运行，不依赖主模型轮询 |
| 保留决策质量 | 异常/失败仍由主模型分析和决策 |

### 2.2 非目标

- 不做多 agent 讨论/协作
- 不做 agent 间的复杂编排
- 不替代主模型的架构决策能力
- 不做实时流式结果回传（结果在任务完成后一次性返回）

## 3. 用户场景

### 3.1 可自动下放的场景（插件硬拦截）

| 场景 | 示例命令 | 预期行为 |
|------|----------|----------|
| 运行测试 | `npm test`, `pytest`, `go test` | 下放给 worker，返回测试摘要 |
| 项目构建 | `npm run build`, `cargo build`, `make` | 下放给 worker，返回构建结果 |
| 日志监控 | `tail -f app.log`, `docker logs -f` | 下放给 worker，运行指定时长后返回异常摘要 |
| 长运行开发服务 | `npm run dev`, `webpack --watch` | 下放给 worker，检测启动成功/失败 |
| 重搜索 | `grep -r "pattern" /path`, `rg` | 下放给 worker，返回匹配结果摘要 |

### 3.2 需要主模型判断的场景（Skill 引导）

| 场景 | 示例 | 预期行为 |
|------|------|----------|
| 脚本执行 | 执行自定义脚本，输出可能很大 | 主模型判断后使用 delegate_task 下放 |
| 批量处理 | 批量重命名、批量格式转换 | 主模型判断后下放 |
| 数据筛选 | 从大量日志中提取特定信息 | 主模型判断后下放 |
| 条件等待 | 等待某个文件出现、等待服务就绪 | 主模型判断后下放 |

### 3.3 不应下放的场景

| 场景 | 原因 |
|------|------|
| 架构决策 | 需要深度理解和创造力 |
| 异常诊断 | 需要主模型分析失败原因 |
| 与用户交互 | 需要理解用户意图 |
| 快速命令 | `ls`, `cat`, `git status` 等秒级完成的命令 |
| 代码编辑 | 需要理解代码语义 |

## 4. 行为规范

### 4.1 主模型行为

- 接到任务后，首先判断当前阶段是否适合下放
- 如果下放，只传给 subagent 完成任务所需的局部信息
- 下放后进入阻塞状态，不持续思考，不轮询
- 收到结果后继续做高价值判断

### 4.2 插件行为

- 拦截明显的长任务命令，自动触发下放
- 估算输出量和耗时，超过阈值则下放
- 注入执行约束到 subagent prompt（最大步骤、输出限制、异常检测规则）
- 过滤 subagent 返回的长输出，提取摘要

### 4.3 Worker 行为

- 使用低级模型独立运行
- 执行命令，观察输出
- 检测异常（错误、警告、非零退出码）
- 可在任务约束内探索代码库（grep/glob/read），服务于搜索、筛选、摘要
- 不做架构决策、不做异常根因判断、不选择修复方案、不与用户交互
- 结果/异常一次性返回
- 不做代码编辑，不再派发子任务

## 5. 约束

| 约束 | 说明 |
|------|------|
| 主模型挂起期零 token 消耗 | 前台模式下，主模型 LLM 调用阻塞，不产生 token |
| 不轮询 | 主模型不主动检查 subagent 状态 |
| 上下文隔离 | subagent 不继承父会话历史，只接收任务 prompt |
| 一次性执行 | subagent 完成后销毁，不保持长连接 |
| 权限继承 | subagent 继承父级权限限制，不能越权 |
```
